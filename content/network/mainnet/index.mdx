# Mainnet Overview

<Callout type="info">
**Recommended**: Follow the quickstart guide [Installation](/network/mainnet/installation) and go through the onboarding using the script.
</Callout>

<Callout type="warning">
Before starting onboarding, make sure you have provided your **Mainnet Ethereum Public Address** to Fermah for whitelisting. Reach out through our [support channels](/network/resources/support) if you haven't done so yet.
</Callout>

For complete manual onboarding, you may follow these steps to get started:

### 1. Installation

Download one of our releases and get your environment setup.

### 2. Configuration

Customize your installation based on your hardware and provers.

### 3. Machine Secret

Setup proper authentication for your prover node.

### 4. Registration

Register your Prover Node with Fermah Core.

### 5. Telemetry

Prepare your node for proving metrics collection.

### 6. Starting a Node

Launch the <GlossaryTooltip>Prover Node</GlossaryTooltip>.

These steps will get a Prover Node fully onboarded into Fermah Froben.

---

### Prerequisites

**Registered Prover Node Accounts:** Ensure you have a fully registered Prover Node account, such as with <GlossaryTooltip>EigenLayer</GlossaryTooltip>. If you don't have one, follow the steps in the [EigenLayer Operator Guide](https://docs.eigenlayer.xyz/operators/howto/operator-installation) to create and fund your account.

You will also need the following:

- Ubuntu 22.04 LTS or Ubuntu 24.04 LTS
- Root Docker v24+ (telemetry — requires host metrics access)
- Rootless Docker v24+ (prover containers)
- <GlossaryTooltip>CUDA</GlossaryTooltip> drivers v12.2 or v12.9 (see NVIDIA [docs](https://developer.nvidia.com/cuda-downloads))
- CUDA containers toolkit for rootless docker (see NVIDIA [docs](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/latest/install-guide.html))
- Mainnet Ethereum Node RPC URL access with your API key (e.g. Alchemy, Infura or private)

<Callout type="info">
Other Linux distributions are also supported as long as they provide the required GLIBC version: **GLIBC 2.34** for CUDA 12.2 and **GLIBC 2.39** for CUDA 12.9.
</Callout>

### Hardware Requirements

Resources used by the Operator will vary greatly depending on the prover image that is running.

#### ZkSync

Minimum requirements for a ZkSync CPU **witness generator**.

| Round | RAM | vCPUs |
| --- | --- | --- |
| all | 64 GB | 16 |
| basic\_circuits | 32 GB | 16 |
| leaf\_aggregation | 80 GB | 16 |
| node\_aggregation | 4 GB | 8 |
| recursion\_tip | 1 GB | 4 |
| scheduler | 1 GB | 4 |

Suggested Cloud Instance Types:
- **AWS** `g6.8xlarge`
- **GCP** `g2-standard-16`

Minimum requirements for a ZkSync **circuit prover.**

| vCPUs | RAM | VRAM | Storage |
| --- | --- | --- | --- |
| 16 | 64 GB | 24 GB | 375 GB SSD |

Suggested Cloud Instance Types:
- **AWS** `g6.8xlarge`
- **GCP** `g2-standard-16`
