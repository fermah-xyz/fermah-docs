# Configuration

<GlossaryTooltip>Prover Node</GlossaryTooltip>s have pre-made configuration files available based on the information provided through the application form.

In order to download these configuration files, you can get them using our Install Script from [installation](/network/mainnet/for-prover-nodes/installation).

<Callout type="info">
During installation, make sure you provide your assigned prover node name, so it fetches the right configuration file.
</Callout>

---

## Notes

Running the Install Script sets up a configuration file corresponding to your prover node, containing information about:

- Network endpoints and contracts
- Keystore
- Hardware resource claims and prices
- Prover capabilities
- Telemetry config

The <GlossaryTooltip>TOML</GlossaryTooltip> configuration file is expected to exist in `~/.fermah/config/prover-node-config.toml`.

<Callout type="info">
For Mainnet, we are currently handling the creation of these configurations from our side for smooth onboarding.
</Callout>

### Example Configuration

<details>
<summary>`~/.fermah/config/prover-node-config.toml`</summary>

The following fields should be customized for each prover node:

- All `[hardware]` entries must reflect the server HW resources.
- Prover fields such as `[zksync]`
- Key name that was setup during [Machine Secret](/network/mainnet/for-prover-nodes/machine-secret)

```toml
[prover_node_machine]
name = "fermah-prover-node-1"
pn_machine_addr = "35.222.190.25:30089"
fermah_runtime_addr = "35.222.190.25:30088"
secret = ""

[keystore]
key_name = "prover-node"

[contracts.avs]
fermahServiceManager = "0xEE4eE799e09Ea975139c93Da2E403a8A0Ea8eE6a"
fermahStakeRegistry = "0x0118abd80cC8895Cbd40f5912e8E36c863CC61c1"

[contracts.el]

[contracts.fermah]

whitelist = "0x1DE8df315db89bF435A278cbA3518f26974d5e1c"

[telemetry]
mode = { Alloy = {} }

interval = 30
temporality = { Cumulative = {} }

[telemetry.export]
endpoint = "http://localhost:4317/"

[storage]

[hardware.ram]
```
